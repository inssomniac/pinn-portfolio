# PINN: уравнение Пуассона на [0,1]²

Эксперименты с Physics-Informed Neural Networks. Решаю уравнение Пуассона, сравниваю архитектуры нейронных сетей и методы генерации коллокационных точек.

## Задача

Уравнение Пуассона с граничными условиями Дирихле на $[0,1]^2$:

$$\Delta u = f(x, y), \quad u\big|_{\partial\Omega} = 0$$

Точное решение: $u = \sin(2\pi x)\sin(3\pi y)$, откуда $f = -13\pi^2 u$.

Нейросеть обучается так, чтобы одновременно минимизировать невязку уравнения и невязку граничного условия.

## Часть 1: архитектуры

Сравниваю четыре архитектуры. Гиперпараметры подобраны Optuna, обучение до 50 000 эпох с early stopping по eval loss на отдельном фиксированном наборе точек.

| Архитектура | Описание |
|-------------|----------|
| **Tanh** | MLP с активацией tanh, 2×128 |
| **GELU** | MLP с активацией GELU, 2×128 |
| **SIREN** | sin-активации с параметром $\omega_0 = 11.3$, 3×128 |
| **HardBC** | Tanh-MLP, умноженный на $\varphi = x(1{-}x)y(1{-}y)$: граничное условие выполняется точно, без штрафного слагаемого в лоссе |

Метрика: relative $L_2$ на равномерной сетке $100 \times 100$. По 10 независимых запусков с разными seed.

Результат выглядел странно: HardBC лучший по rel $L_2$ (около $3 \times 10^{-4}$), но худший по loss. Оказалось, всё логично: у HardBC нет BC-слагаемого в функции потерь, оно тождественно ноль по конструкции. Лосс у него содержит только PDE-невязку, поэтому напрямую сравнивать лоссы между архитектурами нельзя.

## Часть 2: методы коллокации

Фиксирую SIREN. Исходная задача была простая: генерировать точки равномерно. В итоге попробовал ещё три адаптивных метода из Wu et al. (2023).

| Метод | Идея |
|-------|------|
| **uniform** | равномерный случайный (baseline) |
| **RAD** | вероятность точки пропорциональна $\varepsilon^k + c\bar\varepsilon^k$, где $\varepsilon = \|r(x)\|$ |
| **RAD-G** | то же самое, но вес считается по норме градиента невязки $\|\nabla_x r(x)\|$ |
| **RAR-G** | набор точек растёт: каждые 1000 эпох добавляем топ-50 по $\|\nabla_x r(x)\|$ |

Один важный момент про early stopping: нельзя мерить лучшее состояние модели по тому же лоссу, на котором обучаешься с RAD, потому что этот лосс считается на смещённом распределении точек. Для честного сравнения early stopping везде по eval loss на фиксированном равномерном наборе.

На этой задаче адаптивные методы не улучшили среднюю точность: решение гладкое, нет зон с локально высокой ошибкой, которые стоило бы уплотнять. RAD и RAD-G сократили разброс по seeds примерно вдвое. RAR-G оказался хуже baseline: растущий набор точек специализируется на первоначально трудных зонах и мешает обобщению.
